## 1. 导航与外壳

固定左侧边栏（宽 w-56）提供全局导航。

| 路由 | 图标 | 标签 |
|------|------|------|
| `/` | Package | 我的skills |
| `/tools` | Wrench | 我的工具 |
| `/sync/push` | ArrowUpFromLine | 推送到工具 |
| `/sync/pull` | ArrowDownToLine | 从工具拉取 |
| `/starred` | Star | 仓库收藏 |
| `/backup` | Cloud | 云备份 |
| `/settings` | Settings | 设置 |

- 当前路由：使用当前主题色的浅色块高亮，并配合克制的描边与层次阴影。
- 非当前路由：灰色文字，鼠标悬停时显示高亮。
- 侧边栏右上角：**Languages** 语言快捷按钮，可在 **中文** 与 **English** 之间即时切换，并将偏好持久化到 `localStorage`。
- 侧边栏右上角：**调色盘** 主题快捷按钮，按 **Dark → Young → Light** 顺序即时轮换。
- 左下角 **意见反馈** 按钮：在默认浏览器中打开 GitHub 新建 Issue 页面。
- 窗口关闭行为：点击左上角关闭按钮时仅隐藏主窗口，应用继续在后台运行。
- macOS 托盘行为：应用启动后会在顶部状态栏创建黑白状态栏图标；主窗口隐藏后会同时移除 Dock 图标，仅保留该状态栏图标。单击状态栏菜单可执行 `Show SkillFlow`、`Hide SkillFlow` 和 `Quit SkillFlow`。
- Windows 托盘行为：应用驻留在系统托管区，并显示应用自身图标；点击托盘图标会弹出菜单，支持 `Show SkillFlow` 和 `Exit`。

## 2. 我的skills（主面板）

集中管理你的 Skill 库。

### 工具栏

| 控件 | 操作 |
|------|------|
| **搜索框** | 加宽显示，实时按技能名过滤（不区分大小写）；窗口较窄时工具栏会自动换行，保证按钮完整可见 |
| **排序切换** | 两个按钮在 **A-Z** 与 **Z-A** 间切换，按 Skill 名称首字母正序或逆序排列 |
| **更新**（RefreshCw） | 调用后端 `CheckUpdates()`；有更新的 Skill 卡片右上角显示红点 |
| **批量删除**（CheckSquare） | 切换多选模式 |
| **导入**（FolderOpen） | 打开系统文件夹选择器 → `ImportLocal(dir)` |
| **远程安装**（Github） | 打开 GitHub 安装对话框 |

### 多选模式（点击"批量删除"激活）

| 控件 | 操作 |
|------|------|
| **全选 / 取消全选** | 切换当前过滤后所有 Skill 的选中状态 |
| **删除（n 个）**（Trash2，红色） | `DeleteSkills(ids)` — 未选择时置灰禁用 |
| **取消** | 退出多选模式并清空选择 |

### 分类侧边栏

- 列出所有分类；点击可过滤 Skill 网格。
- **"全部" 按钮** — 显示所有分类的 Skills。
- **拖拽目标区域** — 将 Skill 卡片拖到某个分类，即可移动至该分类。
- **拖拽高亮** — 拖动 Skill 卡片悬停到分类上方时，目标分类会以更明显的高亮状态显示。
- **右键上下文菜单**（每个分类）：
  - **重命名** — 显示内联文字输入框；回车确认，Esc 取消；调用 `RenameCategory()`。（`Default` 分类不可用）
  - **删除** — 空分类会直接调用 `DeleteCategory()` 删除；若该分类下仍有 Skill，则弹出提示框要求先清空该分类后再删除。（`Default` 分类仍不可删除）
- **新建分类**（底部 Plus 图标） — 显示内联文字输入框；回车或失焦确认，Esc 取消；调用 `CreateCategory()`。

### Skill 网格

- 布局：3 列，宽屏 4 列。
- **空状态** — 显示"没有找到 Skills"及使用提示。
- **Skill 右键菜单** — 包含"移动到其他分类"项（仅显示非当前分类）、删除，以及适用时的更新操作。
- **拖拽操作** — 将 Skill 卡片拖至侧边栏分类可移动；开始拖拽后，会有一个缩小版卡片跟随鼠标移动；命中左侧分类后，原卡片会收缩成一条细线，直到结束拖拽；将文件夹从操作系统文件管理器拖入窗口可直接导入。
- **窗口级拖拽遮罩** — 拖拽文件进入窗口时，显示半透明靛蓝色遮罩和"松开以导入 Skill"提示。
- **悬浮提示** — 鼠标悬停卡片 300 毫秒后出现（详见 [Skill 悬浮提示](#9-skill-悬浮提示)）。

## 3. 推送到工具

将 Skill 库中的 Skills 复制到外部工具目录。

### 布局

- 采用与"我的skills"接近的双栏布局。
- 左侧边栏显示分类筛选：**全部** 加所有现有分类。
- 右侧显示目标工具选择、搜索 + A-Z/Z-A 排序控件、推送模式按钮，以及当前分类范围内的 Skill 卡片网格。

### 目标工具选择

- 每个已启用的工具对应一个切换按钮（图标 + 名称）。
- 可同时选择多个工具。

### 同步范围

推送行为由左侧分类筛选和右侧模式按钮共同决定：

| 模式 | 行为 |
|------|------|
| **推送全部 / 推送当前分类** | 左侧选中 **全部** 时推送整个库；选中具体分类时只推送该分类 |
| **手动选择** | 以当前左侧分类范围作为候选列表，在卡片上逐个勾选，也可对当前列表执行全选 |

### 目录缺失检查

推送前，应用调用 `CheckMissingPushDirs()`。若目标工具目录尚未创建，弹出确认对话框：

- 列出每个缺失的工具名称及完整目录路径。
- **"创建并推送"** — 创建目录后继续推送。
- **"取消"** — 中止，不创建任何目录。

### 冲突处理

若目标目录中已存在同名 Skill，逐条弹出冲突对话框（详见 [冲突对话框](#101-冲突对话框)）。

### 底部操作栏

- **"开始推送（n）"** 按钮 — 未选工具或数量为零时禁用；进行中显示"推送中…"。
- **"推送完成 ✓"** — 推送结束后绿色成功提示。

## 4. 从工具拉取

将外部工具目录中的 Skills 导入到你的库中。

### 布局

- 采用与“推送到工具”一致的双栏布局。
- 左侧边栏列出全部分类，并直接作为导入目标分类选择器。
- 右侧显示来源工具选择、扫描反馈、搜索 + A-Z/Z-A 排序控件、可选 Skill 网格和底部执行栏。

### 来源工具选择

- 与推送页相同的切换按钮；切换工具时重置已扫描列表。

### 扫描

- **"扫描"** 按钮 — 调用 `ScanToolSkills(toolName)`，递归搜索该工具已配置的扫描目录中的 `skill.md` 文件。
- 本地工具扫描会复用 **设置 → 通用** 中相同的深度限制（默认 `5`，保存范围 `1-20`）。
- 扫描中显示动效"扫描中…"。
- 扫描失败时显示**红色错误提示**；未发现 Skill 时显示**黄色警告提示**。

### Skill 网格

- 扫描成功后显示。
- 搜索框会实时按名称过滤扫描结果。
- 排序切换按钮可在 **A-Z** 与 **Z-A** 间切换。
- 卡片显示该 Skill 是否已导入（绿色"已导入"徽章）。
- 可逐个选择，或对当前可见列表执行"全选 / 取消全选"。

### 底部操作栏

- **"开始拉取（n）"** 按钮 — 调用 `PullFromTool()`。
- **"拉取完成 ✓"** — 绿色成功提示。
- 冲突处理同 [冲突对话框](#101-冲突对话框)。

## 5. 仓库收藏

直接浏览并导入已关注 Git 仓库中的 Skills，无需先导入到库中。

- 仓库扫描会递归遍历整个克隆目录，因此 `plugins/<plugin>/skills/<name>` 这类嵌套 Skill 目录也会被发现；`skill.md` 文件名按大小写不敏感匹配。
- 仓库递归扫描会受到 **设置 → 通用** 中 **远程仓库递归扫描深度** 控件的限制（默认 `5`，保存范围 `1-20`）。

### 视图模式

| 模式 | 图标 | 描述 |
|------|------|------|
| **文件夹视图** | Folder | 仓库卡片网格；点击卡片进入该仓库的 Skill 列表 |
| **平铺视图** | LayoutGrid | 所有仓库的 Skills 合并显示在一个网格中 |

### 工具栏（普通模式）

| 按钮 | 操作 |
|------|------|
| **搜索框** | 在平铺视图和仓库详情视图中显示；按名称过滤当前 Skill 网格 |
| **排序切换** | 在平铺视图和仓库详情视图中显示；在 **A-Z** 与 **Z-A** 间切换 |
| **批量导入**（CheckSquare） | 仅在有 Skill 网格可见时进入多选模式 |
| **全部更新**（RefreshCw） | `UpdateAllStarredRepos()` — 并行克隆/拉取所有仓库；更新中图标旋转 |
| **添加仓库**（Plus，靛蓝色） | 打开"添加仓库"对话框 |

### 工具栏（多选模式）

| 按钮 | 操作 |
|------|------|
| **全选 / 取消全选** | 切换所有可见 Skills |
| **推送到工具（n）** | 打开推送到工具对话框（见下文） |
| **导入到我的skills（n）** | 打开导入对话框 |
| **取消** | 退出多选模式 |

### 仓库卡片（文件夹视图）

- 点击打开该仓库的 Skill 列表。
- **在浏览器中打开**（ExternalLink 图标） — 在默认浏览器中打开仓库 URL。
- **更新**（RefreshCw 图标） — `UpdateStarredRepo(url)` — 拉取最新提交。
- **删除**（Trash2 图标，悬停变红） — 从收藏列表中移除。
- 卡片下方显示最后同步时间及同步错误信息。

### 仓库详情视图（钻取）

- 顶部面包屑返回按钮（ChevronLeft）回到仓库列表。
- Skill 网格与平铺视图的选择/导入操作相同。

### 添加仓库对话框

- URL 输入框（HTTPS 或 SSH 格式）；回车触发添加。
- **"添加"** 按钮 — `AddStarredRepo(url)`。
- 若仓库需要 HTTP 认证，自动弹出 **HTTP 认证对话框**。
- 若 SSH 认证失败，弹出 **SSH 认证错误对话框**，说明所需配置。
- 显示克隆进度状态（"克隆中…"）。

### HTTP 认证对话框

- 用户名 + 密码输入框（密码掩码）；密码框按回车确认。
- **"确认"** — 重试调用 `AddStarredRepoWithCredentials(url, user, pass)`。
- **"取消"** — 中止。
- 凭据错误时显示错误提示。

### SSH 认证错误对话框

- 说明 SSH 密钥配置检查清单：
  - 是否已通过 `ssh-keygen` 生成密钥
  - 公钥是否已添加到 GitHub / GitLab
  - SSH Agent 是否正在运行（`ssh-add`）
  - 建议改用 HTTPS 协议克隆
- **"关闭"** 按钮。

### 导入对话框（到我的skills）

- 分类选择下拉菜单。
- **"导入 n 个"** — `ImportStarSkills(paths, repoURL, category)`。
- **"取消"**。

### 推送到工具对话框

- 说明：将 Skill 直接复制到工具目录，无需先导入到"我的skills"。
- 以复选框列表展示所有已启用工具，右侧显示推送目录路径。
- 无工具时显示提示信息。
- **"推送到 n 个工具"** 按钮。
- **"取消"**。

### 目录缺失确认

与[推送到工具页](#目录缺失检查)行为相同：在创建缺失推送目录前弹框确认。

### 推送冲突对话框

当 Skill 在目标工具目录中已存在时：

- 列出所有冲突的 Skill 名称。
- **"覆盖全部"**（琥珀色） — `PushStarSkillsToToolsForce()`。
- **"跳过冲突"** — 跳过已存在的，仅推送新的。

## 6. 云端备份

将 Skill 库镜像备份至云端。支持两种后端：**对象存储**（阿里云 OSS、腾讯云 COS、华为云 OBS）和 **Git 仓库**。

### 状态提示

- **云备份未启用提示**（黄色横幅） — 未配置时显示，引导前往设置。

### 操作

| 按钮 | 对象存储标签 | Git 标签 |
|------|------------|---------|
| **立即备份**（Upload 图标） | 立即备份 | 立即备份 |
| **恢复 / 拉取**（Download 图标） | 从云端恢复 | 拉取远端 |
| **刷新**（RefreshCw） | 重新加载文件列表 | 同左 |

- 云备份未配置时，两个操作按钮均禁用。
- **"备份完成 / Git 同步完成"**（绿色）/ **"备份/同步失败"**（红色）状态消息。

### 云端文件列表

- 对象存储：文件路径（等宽字体）+ 文件大小（KB）。
- Git：通过 `git ls-files` 列出的跟踪文件，显示相对路径和文件大小。
- 对象存储列表会在内部自动翻页，确保 UI 展示的是完整远端文件集合，而不只是第一页。
- 可滚动容器（最大高度限制）。
- **统一备份范围（所有服务商）** — 备份根目录为应用数据根目录（`skills/`、`meta/`、`config.json` 等）；`cache/` 与 `.git/` 会被排除。
- **同步路径可移植** — `meta/*.json`、`star_repos.json` 等同步元数据中的本地路径会以同步根目录下的正斜杠相对路径保存，确保 macOS 与 Windows 间恢复后仍能正确定位。
- **本地路径配置隔离** — `config_local.json` 保存机器相关的文件系统路径，例如应用目录外的 `SkillsStorageDir` 以及工具 `ScanDirs` / `PushDir`；该文件不参与备份和 git 同步。
- **Git 备份兼容处理** — 当 Git 备份以父目录作为工作树时，SkillFlow 会自动迁移旧的 `skills/.git` 嵌套元数据，避免真实 Skill 文件被当作嵌套仓库而无法跟踪。

### 自动备份

以下操作完成后（在云备份已启用时）自动触发：

- 删除 Skill / 批量删除
- 手动导入
- 从 GitHub 安装
- 从工具拉取
- 更新 Skill
- 从收藏仓库导入

通过 Wails 事件系统向前端推送进度事件：`backup.started`、`backup.progress`、`backup.completed`、`backup.failed`。

### Git 同步（仅 git 服务商）

选择 **git** 服务商后：

- **仓库自举初始化** — 若 Skills 目录还不是 Git 仓库，SkillFlow 会自动执行初始化，并按配置仓库地址设置 `origin`。
- **远端绑定自动修复** — 若 `origin` 缺失或地址变更，拉取/推送前会自动补齐或更新。
- **启动时自动拉取** — 每次 App 启动时，自动对 Git 备份根目录执行 `git pull`，拉取远端最新内容。
- **远端分支缺失容错** — 首次配置时若远端分支尚未创建，启动拉取会跳过而非直接报错。
- **变更后自动推送** — 与对象存储相同的触发时机；执行 `git add -A && git commit && git push`。
- **定时自动同步** — 由"定时自动同步间隔"设置控制（单位：分钟；0 表示禁用）。后台定时器按配置间隔调用 `autoBackup()`。
- **手动操作冲突检测** — 点击 **立即备份** 与 **拉取远端** 时都会检测 Git 冲突/分叉，若需要用户决策会触发 `git.conflict` 事件。
- **冲突解决对话框** — 若 `git pull` 或 `git push` 检测到冲突或历史分叉，弹出模态框：
  - 对话框会在可用时展示冲突相关文件列表。
  - **"以本地为准"** — 中止合并，将本地状态强制推送到远端。调用 `ResolveGitConflict(true)`。
  - **"以远端为准"** — 中止合并，将本地重置为 `origin/<分支>`。调用 `ResolveGitConflict(false)`。
  - 两种操作成功后均从磁盘重载应用状态（skills/meta/config），并发送 `git.sync.completed` 事件。
- **拉取后即时刷新** — 启动自动拉取或手动拉取成功后，会立刻重载磁盘状态，使 `meta/` 与配置变更立即生效。
- 若冲突发生在启动阶段（前端尚未加载），将以待处理标志存储，并在备份页面挂载时通过 `GetGitConflictPending()` 弹出对话框。

## 7. 设置

配置面板，分四个标签页。

### 工具路径标签页

为每个内置或自定义工具提供以下配置：

| 控件 | 说明 |
|------|------|
| **启用开关** | 在整个应用中启用或禁用该工具 |
| **推送路径** | 推送时复制 Skill 的目标目录（唯一）；支持直接输入文字和点击文件夹图标（FolderOpen）打开系统文件管理器选择，并会定位到当前路径或最近存在的父目录 |

## 11. 后端事件

Go 后端通过 Wails runtime 向前端推送的事件：

| 事件 | 触发时机 | 载荷 |
|------|---------|------|
| `backup.started` | 自动备份开始 | — |
| `backup.progress` | 每上传一个文件 | `{ currentFile: string }` |
| `backup.completed` | 备份完成 | — |
| `backup.failed` | 备份出错 | — |
| `update.available` | 发现 Skill 有新提交 | `{ skillID, skillName, currentSHA, latestSHA }` |
| `star.sync.progress` | 单个仓库同步完成 | `{ repoURL, repoName, syncError }` |
| `star.sync.done` | 所有仓库同步完成 | — |
| `git.sync.started` | Git 拉取/推送开始 | — |
| `git.sync.completed` | Git 同步成功 | — |
| `git.sync.failed` | Git 同步出错 | — |
| `git.conflict` | 检测到 Git 合并冲突 | `{ message: string, files?: string[] }` |

主面板监听 `update.available` 事件，实时在对应 Skill 卡片上标记红色更新红点。
备份页监听所有 `git.*` 事件，并在收到 `git.conflict` 时弹出冲突解决对话框。
`App.tsx` 监听全部三个 `app.update.*` 事件，解析发出的 `AppUpdateInfo` payload，并驱动更新对话框的状态机。

## 12. 应用更新对话框

检测到新版本时弹出的居中模态对话框，同时适用于启动时自动检测和设置页手动检测两个入口。对话框由四态状态机驱动：

| 状态 | 触发条件 | 对话框内容 |
|------|---------|----------|
| `available` | 收到 `app.update.available` 事件 | 版本号 + 更新说明 + 平台对应操作按钮 |
| `downloading` | 用户点击"下载并自动重启更新"（仅 Windows） | 旋转图标 + 下载进度提示 |
| `ready_to_restart` | 收到 `app.update.download.done` 事件 | 完成提示 + "立即重启" / "稍后重启"按钮 |
| `download_failed` | 收到 `app.update.download.fail` 事件 | 错误提示 + "前往下载页"按钮 |

### 平台行为

启动检测与手动检测均使用同一模态对话框。

- **Windows** — `available` 状态提供三个选项：
  1. **下载并自动重启更新** — 后台下载新 exe，完成后提示重启。
  2. **前往 Release 页面手动下载** — 在系统浏览器打开 GitHub Releases 页面。
  3. **跳过此版本（下次启动不再提示）** — 记住跳过的版本；启动检测不再对该版本弹窗，手动检测不受影响，仍正常弹出。
- **macOS** — `available` 状态提供两个选项（暂不支持自动下载）：
  1. **前往 Release 页面手动下载** — 在系统浏览器打开 GitHub Releases 页面。
  2. **跳过此版本（下次启动不再提示）** — 与 Windows 行为一致。
- 两个平台都使用同一份 `AppUpdateInfo` payload 渲染当前版本、最新版本和 Release 页面入口。

### 跳过版本机制

- 跳过的版本 tag 存储在 `AppConfig.SkippedUpdateVersion`，并写入共享的 `config.json`，因此重启应用后仍会生效。
- 启动检测时，若 `latestVersion == skippedUpdateVersion`，则**不**发布 `app.update.available` 事件，不弹出对话框。
- 设置页手动点击"检测更新"时，`CheckAppUpdateAndNotify` 始终发布事件，不受跳过版本影响 — 手动检测总是正常弹出对话框。
- 点击"跳过此版本"时调用 `SetSkippedUpdateVersion(latestVersion)`。

### 手动检测按钮（设置页）

设置页标题栏右上角提供 **"检测更新"** 按钮：

- 按钮旁显示当前应用版本号（`vX.Y.Z`）。
- 点击 → 调用 `CheckAppUpdateAndNotify()`；检测中按钮显示旋转图标。
- 若发现新版本，更新对话框通过 `app.update.available` 事件自动弹出。
- 若已是最新版本，内联显示"已是最新版本 (vX.Y.Z)"。
- 出错时内联显示"检测失败: …"。

### 操作说明

| 控件 | 操作 |
|------|------|
| **下载并自动重启更新**（Windows，`available` 状态） | `DownloadAppUpdate(downloadUrl)` — 启动后台异步下载 |
| **前往 Release 页面手动下载**（`available` 状态） | `OpenURL(releaseUrl)` — 在浏览器打开发布页；关闭对话框 |
| **跳过此版本**（`available` 状态） | `SetSkippedUpdateVersion(latestVersion)` — 持久化跳过版本；关闭对话框 |
| **立即重启**（`ready_to_restart` 状态） | `ApplyAppUpdate()` — 写入批处理脚本并退出；脚本完成 exe 替换后重启 |
| **稍后重启**（`ready_to_restart` 状态） | 关闭对话框，不立即重启 |
| **前往下载页**（`download_failed` 状态） | `OpenURL(releaseUrl)` — 在浏览器打开发布页；关闭对话框 |
| **×**（除 `downloading` 外所有状态） | 关闭对话框（本次会话内不再提示） |

### 后端方法

| 方法 | 说明 |
|------|------|
| `GetAppVersion()` | 返回当前版本字符串（CI 通过 `-ldflags` 注入；本地开发为 `"dev"`） |
| `CheckAppUpdate()` | 查询 GitHub Releases API；返回含平台匹配下载链接的 `AppUpdateInfo` |
| `CheckAppUpdateAndNotify()` | 调用 `CheckAppUpdate()` 并在有更新时发布 `EventAppUpdateAvailable`；始终通知，不受跳过版本影响 |
| `GetSkippedUpdateVersion()` | 返回 config 中存储的用户跳过版本 tag |
| `SetSkippedUpdateVersion(version)` | 将跳过版本 tag 持久化到 config；传空字符串可清除 |
| `DownloadAppUpdate(url)` | 异步下载新 exe 到临时目录；完成后发布 `app.update.download.done` 或 `app.update.download.fail` 事件 |
| `ApplyAppUpdate()` | 仅 Windows — 写入批处理脚本实现退出后 exe 替换，然后调用 `os.Exit(0)` |

### 版本注入（CI）

GitHub Actions 在编译时通过 ldflags 注入 tag 名：
```
wails build -ldflags "-X main.Version=${{ github.ref_name }}"
```
当 `Version == "dev"` 时（本地开发），启动时的更新检测会跳过。
