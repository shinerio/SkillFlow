│                                                                              │
│ - Wails build 产出 .app bundle，可直接 zip 分发                              │

│ Windows                                                                      │
│                                                                              │
│ - Wails build 产出单个 .exe                                                  │
│ - Windows 11 / 大多数 Win10 已内置 WebView2（Wails v2                        │
│ 依赖），无需安装即可运行                                                     │
│ - 极少数旧系统缺 WebView2，可在 README 提示手动安装；或用 -nsis flag         │
│ 打包成有 WebView2 引导的安装包（可选）                                       │
│                                                                              │
│ 结论：可以实现免安装分发。macOS 必须 zip（.app 是目录 bundle），Windows      │

│ // GetAppVersion 返回当前版本号（由 ldflags 注入）                           │
│ func (a *App) GetAppVersion() string                                         │
│                                                                              │
│ // CheckAppUpdate 查询 GitHub Releases API，返回更新信息                     │
│ // 根据 runtime.GOOS + runtime.GOARCH 匹配对应平台的 asset 下载 URL          │
│ func (a *App) CheckAppUpdate() (*AppUpdateInfo, error)                       │
│                                                                              │
│ // DownloadAppUpdate 后台下载新版本到临时目录，通过事件上报进度              │
│ // Windows only: 下载完成后 emit EventAppUpdateDownloadDone                  │
│ func (a *App) DownloadAppUpdate(downloadURL string) error                    │
│                                                                              │
│ // ApplyAppUpdate Windows only: 写 bat 脚本实现替换并重启，然后 os.Exit(0)   │
│ // bat 脚本逻辑：timeout 2 → move new_exe current_exe → start current_exe →  │
│ del self                                                                     │
│ func (a *App) ApplyAppUpdate() error                                         │
│                                                                              │
│ CheckAppUpdate 实现细节：                                                    │
│ - 调用 https://api.github.com/repos/{owner}/{repo}/releases/latest           │
│ - 对比 tag_name 与 Version（字符串比较，tag 格式 v1.2.3）                    │
│ - 从 assets 数组中按命名规则匹配当前平台的下载 URL：                         │
│   - windows/amd64 → asset 名含 windows                                       │
│   - darwin/amd64 → asset 名含 macos-intel                                    │
│   - darwin/arm64 → asset 名含 macos-apple-silicon                            │
│ - 复用 a.proxyHTTPClient() 支持代理                                          │
│                                                                              │
│ ApplyAppUpdate bat 脚本内容：                                                │
│ @echo off                                                                    │
│ timeout /t 2 /nobreak > nul                                                  │
│ move /y "<tmp_new_exe>" "<current_exe_path>"                                 │
│ start "" "<current_exe_path>"                                                │
│ del "%~f0"                                                                   │
│ 启动 bat 后调用 os.Exit(0) 退出当前进程，bat 接管完成替换。                  │

❯ github action是否支持通过macos-latest自动打包为.dmg镜像，通过这个镜像实现 
  自动更新是否可行。                                                        

⏺ 技术分析

  CI 打包 DMG：完全可行

  macos-latest 内置 hdiutil，可直接生成 DMG：

  # 创建 "拖拽到 Applications" 风格的 DMG
  hdiutil create -volname "SkillFlow" \
    -srcfolder build/bin/SkillFlow.app \
    -ov -format UDZO \
    SkillFlow-macos-intel.dmg

  CI 改动很小，把 zip 换成 hdiutil 命令即可。

  ---
  用 DMG 实现自动更新：不可行

  这是两个独立问题：

  ┌────────────────┬───────────────────────────────────────┐
  │      场景      │              DMG 的作用               │
  ├────────────────┼───────────────────────────────────────┤
  │ 首次分发安装   │ ✅ 更友好（挂载 → 拖到 Applications） │
  ├────────────────┼───────────────────────────────────────┤
  │ 应用内自动更新 │ ❌ 完全不适合                         │
  └────────────────┴───────────────────────────────────────┘

  为什么 DMG 不能用于自动更新：

  1. DMG 是磁盘镜像，需要用户手动操作：挂载 → 拖拽 → 推出，无法静默完成
  2. .app 是目录 bundle，无法简单替换：Windows 的 bat 脚本替换 exe 的方案在
  macOS 不适用——.app 是文件夹，正在运行时系统会锁定部分文件
  3. Gatekeeper 拦截：即使静默下载替换了 .app，新版本会因为来源未知被 Gatekeeper
   拦截（需要右键→打开）
